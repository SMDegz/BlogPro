1、云开发基础

发表于 2023-05-01 更新于 2024-01-27 阅读次数： Valine：

笔记

总结

1、并发量增大，数据量增大，业务数据量上升，

开发员上升，业务复杂性上升。

选择：微服务架构解决。

开发上云的思路：

前提：

​ 1、大项目 2、用云才划算。

​ 2、小项目 用云不划算。

总结

1、分布式项目：直接使用云平台技术（云开发）

2、单体项目：本地开发

全部上云缺陷：

Docker k8s Linux 全部是别人提供。

1、数据安全问题。

2、公司成本可能不会降低很多问题。

私有云方案：

1、Docker / k8s。这两种技术。

云—->无限：Docker /k8s 可以部署无数个实例。

现在比较成熟。

技术：全部自己维护。经济成本最高

​ 所有技术全部自己做。

公有云：全部交给别人。经济成本其次

私有云：自己维护+云平台维护一部分。最经济

​ 利用云平台提供的Linux，自己维护Docker k8s。

​ 云主机+Docker +k8s私有云。

回顾：

1、开发为什么上云。

2、开发如何上云？

使用：云平台。

1、Azure讲解。可以付费的时候。再进行讲解。 第二期的时候，有录播。

微软MVP Azure云。 MVP。

微软系统架构 MVP。

国内：阿里云：腾讯云。基本使用。就可以。

安排：

1、第二期的讲解。可以先看录播

云平台：花钱找人办事。

核心部分：

1、laaS 基础设施即服务。云主机。

2、PaaS 平台即服务。项目技术。

3、SaaS 软件即服务。自己项目。云主机+sdk环境

扩展部分：

4、FaaS 功能即服务。开发（云主机，环境）云主机+sdk环境（阿里进行了封装）

5、BaaS 区块链即服务。

只是对所有服务的归类。

分布式缓存缺陷是命中率低吗？

缺陷

1、缓存击穿。redis数据过期。

2、缓存奔溃。redis宕机。

两种方案

1、redis集群。

2、多级缓存。

混合云简单的说下

多个云平台合在一起。

Azure+阿里云。

SaaS与FaaS,区别是部署平台不一样吗

Saas是自己部署。

FaaS是别人给你部署。
