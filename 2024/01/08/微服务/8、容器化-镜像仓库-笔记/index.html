5、Harbor 存储.Net7项目镜像

目标：.Net7项目镜像上传到Harbor 私有镜像仓库

阶段：

1、创建.Net7项目镜像

docker build -t 192.168.142.168/ydt_project/product .

2、.Net7项目镜像上传到私有镜像仓库

docker push 192.168.142.168/ydt_project/product

3、Docker中运行 192.168.142.168/ydt_project/product

1、先下载

docker pull 192.168.142.168/ydt_project/product

2、才运行

docker run -d -P 192.168.142.168/ydt_project/product

私有仓库好处：镜像创建一次，多次运行。不需要每个docker节点都去创建一次。

总结：单个docker中运行私有仓库镜像

1、只需要创建一次Net7项目镜像镜像

2、可以在其他节点上进行多次运行

注意： 

1、在192.168.142.129节点中，配置私有仓库地址
{
    "registry-mirrors": ["https://hub-mirror.c.163.com"],
    "insecure-registries":["192.168.142.168"]
}

2、在192.168.142.129节点中，登录仓库地址

	[root@localhost data]# docker login 192.168.142.168
	Username: admin
	Password: Harbor12345
	Login Succeeded

6、Docker集群中 运行Harbor 中镜像

目标：Docker集群中 运行 .Net7项目镜像

docker service create --publish 8034:80 192.168.142.168/ydt_project/product

7、k8s 中运行Harbor 中镜像

目标：k8s集群中 运行 .Net7项目镜像

kubectl run productpod --image=192.168.142.168/ydt_project/product

前提：

1、k8s登录私有镜像仓库

kubectl create secret docker-registry rigistry --docker-server=192.168.142.168 --docker-username=admin --docker-password=Harbor12345

2、使用token：rigistry

工具：product.yml

apiVersion: v1
kind: Pod
metadata:
  name: product
  labels:
    app: product
spec:
  containers:
  - image: 192.168.142.168/ydt_project/product
    name: product-container
    imagePullPolicy: IfNotPresent
    ports:
     - containerPort: 80
  imagePullSecrets: #使用rigistry
       - name: rigistry

 

 

1、不加tag，上传同名镜像会覆盖吗？

答案：会覆盖

2、k8s获取habor的token之后，这个token不会过期吗？会不会过几天就又要登录了？

答案：不会过期

3、token可以修改吗？

答案：可以修改

kubectl create secret docker-registry rigistry --docker-server=192.168.142.168 --docker-username=admin --docker-password=Harbor12345

kubectl update secret docker-registry rigistry --docker-server=192.168.142.168 --docker-username=admin --docker-password=Harbor12345

4、k8s获取token，是从docker哪里获取的吗？docker必须登录

答案：可docker没有关系，属于k8s登录。
