MVC的四种传值方式

  • ViewData:

    ViewData 是一个字典对象，它允许你将数据从控制器传递到视图。
    它通常用于传递简单的数据，如字符串、数字等。
    在控制器中，你可以这样设置 ViewData: ViewData[“Key”] = value;
    在视图中，你可以这样获取数据: var value = ViewData[“Key”];

  • ViewBag:

    ViewBag 是一个动态类型对象，它允许你从控制器向视图传递数据。
    它提供了一种更加动态的方式来设置数据，这意味着你可以在运行时决定属性的名称和值。
    在控制器中，你可以这样设置 ViewBag: ViewBag.Key = value;
    在视图中，你可以这样获取数据: var value = ViewBag.Key;

  • Model:

    Model 是视图的核心概念之一，它允许你传递一个对象或对象的集合到视图。
    这是最常见的方法来传递复杂的数据结构，如实体、DTOs、ViewModel等。
    在控制器中，你可以这样设置 Model: return View(myModel);
    在视图中，你可以这样使用模型: @model MyModelType 和通过模型属性访问数据。

  • TempData:

    TempData 是一个短暂的存储机制，它允许你在同一个请求的后续阶段（例如从控制器到同一个或另一个视图）传递数据。
    它使用 session 机制来存储数据，但当数据被读取后会自动从 session 中删除。
    这使得 TempData 非常适合用于传递一些需要在两个或更多视图之间共享的数据。
    使用方法：在控制器中设置 TempData 属性并传递值，然后在视图中读取它。

PartialPage局部页

“PartialPage” 通常指的是一个网页的部分内容，它可以独立于主页面进行加载、修改或更新。在许多Web开发框架中，例如ASP.NET MVC，partial视图（或partial页面）是一个常见的概念。

使用partial页面有几个好处：

性能优化：如果一个页面的某一部分经常改变，而其他部分保持不变，那么只更新这部分内容而不是整个页面可以大大提高性能。
代码重用：如果多个页面有相同的内容或布局，使用partial页面可以避免重复代码。
更易于维护：如果页面的一部分需要更改，只需更改一个地方，而不是多个地方。

在ASP.NET MVC中，partial视图通常使用@Html.Partial()或@Html.RenderPartial()方法来渲染。
